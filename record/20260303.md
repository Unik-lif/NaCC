## 实验记录

综合前几天的进展，今天的重点是落实 Fork 支持的第一阶段：fork + exec 支持。

### 代码实现（今日完成）

#### Linux 侧

- **`arch/riscv/kernel/process.c` — `copy_thread()`**
  - 添加 `NACC_FORKED` 标记：当父进程是 NACC 保护进程（`NACC_INITED`）时，子进程 `nacc_flag` 设为 `NACC_FORKED`

- **`arch/riscv/include/asm/processor.h` — `thread_struct`**
  - 新增 `unsigned long nacc_cid` 字段，用于在进程结构中直接持有容器 CID
  - fork 时天然继承，子进程无需再查表

- **`arch/riscv/kernel/sys_riscv.c` — `nacc_register()`**
  - 注册时同步写入 `current->thread.nacc_cid = cid`

- **`arch/riscv/kernel/sys_riscv.c` — `nacc_invoke_child()`**
  - 完整实现：分配 VM_NACC VMA、从 `thread.nacc_cid` 读取 CID，调用 SBI ecall
  - **不跳转 agent**，SBI 返回后子进程继续在 Linux 执行

#### OpenSBI 侧

- **`include/sm/sm.h`** — 声明 `sm_nacc_invoke_child(va, pid, cid)`
- **`lib/sbi/sbi_ecall_nacc.c`** — 新增 `SBI_EXT_LINUX_INVOKE_CHILD`（funcid=0x6）dispatch
- **`lib/sbi/sm/sm.c` — `sm_nacc_invoke_child()`**
  - 读取子进程 SATP，搬运 PTP 到安全内存
  - 注册子进程 PID 到 CID（使用 Linux 传递的精确 CID）
  - 调用 `agent_mmap` 映射 agent 区域到子进程页表
  - 设置 `NACC_STATE` 为 `NACC_STATE_LINUX`（子进程留在 Linux 上下文）
  - **不调用 `agent_prepare`**，正常 return
- **`include/sm/cid.h` + `lib/sbi/sm/cid.c` — `nacc_get_first_cid()`**
  - 辅助函数（后续已被更好的方案替代，直接从 Linux 传 CID）

### 测试与发现

#### 测试命令：`docker run busybox sh -c "cat /tmp/test.txt"`

关键发现：
- 系统调用序列中**无 `sys_clone`，只有一次 `sys_execve`**（sh 直接 exec 成 cat，busybox 的 exec 优化）
- 因此**没有触发 fork 流程**，`nacc_invoke_child` 未被调用
- 真正的 Crash 原因是：sh exec 成 cat 时，`begin_new_exec → exit_mmap → free_pgd_range → kmem_cache_free` 试图释放 NaCC 安全内存中的 PTP 页，导致内核 Oops

```
epc : kmem_cache_free+0x8a/0x492   ← 试图释放 reserved 页
 ra : free_pgd_range+0x586/0xde2
```

#### 测试命令：`docker run busybox sh -c "echo hello; echo b; echo c"`

- `echo` 是 busybox sh 内建命令，无 fork 无 execve
- 观察到正常的 3 次 `sys_write` + 正常 `sys_exit_group` 退出
- **整个过程没有 crash**，NaCC 保护在这种场景下完全正常

### 结论与后续计划

**今日发现**：当前阻塞不在 fork 支持，而在 **execve 路径中的 PTP 释放问题**。

详细计划见 `record/nacc_execve_plan.md`，核心下一步：

1. **Phase 0（紧急）**：修复 `exit_mmap` 路径，使 NaCC 的 PTP 页在进程 execve 时能被安全清理，不走 `kmem_cache_free`
2. **Phase 1**：在 execve 修复后，用 `sh -c "cat file; echo done"` 触发真实的 fork+exec 场景，验证 `nacc_invoke_child` 流程
