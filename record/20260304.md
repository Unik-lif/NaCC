## 2026-03-04 实验记录

### 背景
昨日发现当 `NACC_INITED` 进程执行 `execve` 时（如 sh → cat），`exit_mmap` 会触发 `kmem_cache_free` 释放已被 NaCC 迁移到安全内存的 PTP 页，导致内核 Oops。

### 今日完成内容

#### Phase 0: execve 路径修复 ✅

修改文件：`linux/fs/exec.c` (`begin_new_exec` 函数)

**Bug 1: `kmem_cache_free` Oops**
- 原因：`exec_mmap` 释放旧 mm 时，`free_pgtables` 试图普通释放 NaCC 安全内存中的 PTP 页
- 修复：在 `exec_mmap` 之前，若 `nacc_flag & NACC_INITED`，设置 `NACC_RECLAIM` 标志
- 效果：复用 `sys_exit_group` 已有的安全回收路径，7 个 PTP 页正确回收

**Bug 2: `__pte_alloc_one_noprof` BUG**
- 原因：`exec_mmap` 后 `nacc_flag` 仍为 `NACC_INITED|NACC_RECLAIM`，导致 `load_elf_binary` 在新 mm 上走 NaCC PTP 分配路径
- 修复：`exec_mmap` 成功后立刻将 `nacc_flag` 重置为 `NACC_PREPARE`
- 效果：ELF 加载使用普通页表，完成后 `bprm_execve` 末尾检测到 `NACC_PREPARE`，调用 `nacc_invoke()` 重建保护

#### 验证结果
- 测试命令：`docker run busybox sh -c "cat /tmp/test.txt"`
- **结果：完全成功**
  - `NACC_RECLAIM` 正确设置，旧 mm 的 PTP 安全回收 ✅
  - `nacc_flag` 清除为 `NACC_PREPARE`，`load_elf_binary` 正常加载 cat ✅
  - `nacc_invoke()` 重新建立保护，Agent `Hello World` 打印 ✅
  - cat 正常执行（`can't open` 是因为文件不存在，行为正确）✅
  - `sys_exit_group` 正常退出，PTP 再次回收（7页），页表清空 ✅
  - **无 Oops / Kernel panic** ✅
- 残留非致命 Warning：
  - `BUG: Bad rss-counter state` — NaCC PTE 操作未更新 RSS 计数器
  - `BUG: Bad page state pfn:1bfff4` — 已回收的安全内存页被其他进程碰到

#### execve 完整流程

```
sh (NACC_INITED) 执行 execve(cat)
  → begin_new_exec:
    1. nacc_flag |= NACC_RECLAIM
    2. exec_mmap: 释放旧 mm（走 NACC_RECLAIM 安全回收路径）
    3. nacc_flag = NACC_PREPARE（让新 mm 走普通页表）
  → load_elf_binary: 正常加载 cat
  → bprm_execve 返回:
    4. 检测 NACC_PREPARE → nacc_invoke() 重建保护
  → cat 正常运行（在 NaCC 保护下）
  → sys_exit_group: 正常退出
```

### 下一步规划

#### Phase 1: fork + exec 测试

**目标**：验证 `NACC_FORKED` 路径在 fork+exec 场景下是否工作。

**测试命令**：
```bash
docker run busybox sh -c "cat /etc/hostname; echo done"
```

该命令会触发：
```
sh (NACC_INITED)
  ├─ fork() → 子进程 (NACC_FORKED)
  │    └─ execve("cat") → 需处理 NACC_FORKED + execve 的交互
  └─ 父进程 wait()
       └─ echo done（内建命令）
       └─ sys_exit_group
```

**关注点**：
1. fork 时 `NACC_FORKED` 标志是否正确继承
2. 子进程 execve 时是否正确处理 `NACC_FORKED` 状态
3. 父进程 wait 后是否能正常继续
4. 两个进程的 `sys_exit_group` 是否都能正确回收 PTP
